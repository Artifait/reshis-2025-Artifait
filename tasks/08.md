# Задача 8 — Деактивация пользователей

## Контекст

Администратор обнаружил серьезную проблему безопасности: несколько учеников, которые были отчислены из школы, до сих пор имеют доступ к электронному дневнику. 
Это нарушает политику конфиденциальности и создает риски несанкционированного доступа к данным.

Необходимо срочно реализовать функцию деактивации (блокировки) пользователей, доступную только администратору.

## Требования

### Функциональные требования:
- Добавить возможность **деактивации пользователей** (блокировка доступа).
- Добавить возможность **активации пользователей** (восстановление доступа).
- Кнопка управления статусом пользователя должна быть **в навигационном меню** (header).
- **Только администратор** может деактивировать и активировать пользователей.
- Деактивированные пользователи **не могут войти в систему**.

### Требования к интерфейсу:
- В меню хедера добавить пункт **«Управление пользователями»** (видимый только для администратора).
- На странице управления отображать список всех пользователей с их текущим статусом.
- Для каждого пользователя показывать:
  - имя и фамилию;
  - e-mail;
  - роль;
  - текущий статус (Активен/Заблокирован);
  - кнопку изменения статуса.


### Требования безопасности:
- Администратор **не может деактивировать сам себя**.
- При попытке входа деактивированного пользователя показывать сообщение об ошибке.
